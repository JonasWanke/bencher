간편함을 위해 이 튜토리얼에서는 모의 데이터를 사용하겠습니다.

실행: `bencher run --project YOUR_PROJECT_SLUG "bencher mock"`

이제 브라우저에서 각 벤치마크의 결과를 볼 수 있습니다.
`View results`에서 링크를 클릭하거나 복사하여 붙여넣기 하세요.
각 벤치마크에 대한 데이터 포인트는 하나뿐이므로 데이터를 더 추가해 봅시다!

<br />

먼저, 환경 변수로 프로젝트 slug를 설정하여 매번 `--project`를 제공하지 않아도 되게 하겠습니다.

실행: `export BENCHER_PROJECT=save-walter-white-12345`

그런 다음 실행: `echo $BENCHER_PROJECT`

다음과 같이 나와야 합니다:

```
$ echo $BENCHER_PROJECT
save-walter-white-12345
```

<br />

`--project` 없이 같은 명령을 다시 실행하여 더 많은 데이터를 생성해 봅시다.

실행: `bencher run "bencher mock"`

<br />

이번에는 결과를 `bencher run`으로 파이프하여 더 많은 데이터를 생성해 봅시다.

실행: `bencher mock | bencher run`

<br />

결과를 파일에 저장하고 `bencher run`이 이를 가져가게 하려는 경우가 있을 수 있습니다.

실행: `bencher run "bencher mock > results.json" --file results.json`

<br />

또한, 별도의 프로세스가 벤치마크를 실행하고 결과를 파일에 저장하게 하고, `bencher run`이 그것을 가져가게 할 수도 있습니다.

실행: `bencher mock > results.json && bencher run --file results.json`

<br />

마지막으로, `bencher run`의 `--iter` 인수를 사용하여 많은 데이터를 씨앗처럼 심어봅시다.

실행: `bencher run --iter 16 "bencher mock"`

<br />

> 🐰 팁: [`bencher run` CLI Subcommand 문서](/docs/ko/explanation/bencher-run)에서 `bencher run`이 수행할 수 있는 모든 작업에 대해 자세히 알아보세요!

## 알림 생성

이제 벤치마크에 대한 일부 과거 데이터가 있으니, 알림을 생성해 봅시다!
알림은 벤치마크 결과가 성능 저하로 판단될 때 생성됩니다.
그러니 성능 저하를 시뮬레이션해 봅시다!

실행: `bencher run "bencher mock --pow 8"`

<br />

출력의 끝부분에 `View alerts`라는 새로운 부분이 나타나야 합니다:

```
View alerts:
- bencher::mock_0: https://bencher.dev/ ...
- bencher::mock_1: https://bencher.dev/ ...
- bencher::mock_2: https://bencher.dev/ ...
- bencher::mock_3: https://bencher.dev/ ...
- bencher::mock_4: https://bencher.dev/ ...
```

<br />

이제 브라우저에서 각 벤치마크의 알림을 볼 수 있습니다.
`View alerts`에서 링크를 클릭하거나 복사하여 붙여넣기 하세요.

<br />

> 🐰 팁: 성능 저하가 어떻게 감지되는지에 대한 전체 개요는 [임계값 및 알림 문서](/docs/ko/explanation/thresholds)를 확인하세요!

<br />
